Твоя часть (дома):
1. Скачай Docker Desktop если нет
2. Положи все файлы в одну папку
3. Заполни styleguide.md правилами из Confluence
4. docker build -t pr-review-bot .
5. Smoke-тест (без токенов, просто что контейнер жив):
   docker run --rm -p 9000:9000 pr-review-bot
   → http://localhost:9000/health должен ответить
   ВАЖНО: это только проверка живости. Реального ревью тут НЕ будет —
   токенов и настроек нет (см. раздел про ENV ниже). Для боевого запуска — compose.
6. docker save pr-review-bot -o pr-review-bot.tar
7. Передай Ярославу: .tar + docker-compose.yml + .env.example + styleguide.md

Часть Ярослава (виртуалка):
8. docker load -i pr-review-bot.tar
9. Создать .env из .env.example и заполнить реальными токенами
10. docker-compose up -d
11. Настроить реверс-прокси (как у твоего MCP)
12. Настроить webhook в Bitbucket

═══════════════════════════════════════════════════════════════
КАК ЗАПУСКАТЬ ПРАВИЛЬНО (важно!)
═══════════════════════════════════════════════════════════════
Бот читает ВСЮ конфигурацию из переменных окружения (ENV). В коде токенов нет.

Боевой запуск — ТОЛЬКО через docker-compose (НЕ через голый docker run):
  - docker-compose автоматически подхватывает файл .env из той же папки;
  - значения из .env подставляются в docker-compose.yml (${ПЕРЕМЕННАЯ});
  - .env НИКОГДА не пушим в git и не кладём в образ — только на сервере.

Порядок на сервере:
  cp .env.example .env      # один раз
  nano .env                 # вписать реальные токены
  docker-compose up -d      # запуск (читает .env)

Поменять настройку (например, поднять таймаут) БЕЗ пересборки образа:
  nano .env                 # изменить значение
  docker-compose up -d      # перечитает .env и пересоздаст контейнер
(образ пересобирать НЕ нужно — меняется только окружение)

═══════════════════════════════════════════════════════════════
ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ (полный список — в .env.example)
═══════════════════════════════════════════════════════════════
Обязательные (без них бот не сделает ревью):
  BITBUCKET_URL     — адрес Bitbucket
  BITBUCKET_TOKEN   — токен сервисного аккаунта (Repo:Read + PR:Write)
  FENIX_URL         — адрес Феникса
  FENIX_TOKEN       — токен Феникса
  FENIX_MODEL       — модель (по умолчанию deepseek НЕдумающая)

Опциональные — тюнинг (есть дефолты, можно не задавать):
  MAX_DIFF_LINES        =300   максимум строк диффа за запрос
  FENIX_MAX_TOKENS      =4096  лимит длины ответа модели; меньше = быстрее
                               и экономнее по лимиту ~700k ток/мин.
                               Если в логах finish_reason=length — поднять.
  FENIX_TIMEOUT         =90    таймаут запроса к Фениксу (сек).
                               На webhook не влияет (ревью идёт в фоне).
  FENIX_MAX_CONCURRENCY =1     сколько ревью идут к Фениксу разом.
                               1 = по очереди, защита от спайков 700k/мин.
  FENIX_MAX_RETRIES     =1     повторы при таймауте/429 (0 = выкл).
                               Помогает в основном при 429.

Подсказки в логах: при таймауте бот пишет ⏱️, при лимите 700k — 🚦,
и прямо подсказывает, какую переменную крутить.
