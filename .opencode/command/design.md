Спроектировать фичу через цикл architect → critic → утверждение.

Задача: $ARGUMENTS

Шаги:
1. Убедись что есть spec.md для этой фичи. Если нет — попроси меня написать (Spec Kit /specify).
2. Запусти субагента **architect** — пусть спроектирует решение по spec.md, создаст plan.md.
3. Запусти субагента **critic** — пусть оспорит plan.md, найдёт слабые места и риски.
4. Покажи мне: план архитектора + замечания критика рядом.
5. ОСТАНОВИСЬ. Жди моего решения — что из критики учесть.

Не переходи к коду. Это только проектирование. Финальное слово за мной.
