Полный цикл разработки фичи: проектирование + реализация.

Задача: $ARGUMENTS

ЭТАП 1 — Проектирование:
1. субагент **architect** → plan.md по spec.md
2. субагент **critic** → оспаривает план
3. Показать мне план + критику. ЖДАТЬ моего утверждения плана.

ЭТАП 2 — Реализация (только после моего ОК на план):
4. субагент **coder** → реализует
5. субагент **reviewer** → проверяет
6. coder исправляет критичное, reviewer перепроверяет
7. Показать результат. ЖДАТЬ моего финального решения.

Два human checkpoint: после плана и после кода.
Не коммить без моего разрешения.
