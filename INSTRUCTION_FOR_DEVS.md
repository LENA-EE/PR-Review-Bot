# JARVIS PR Review Bot — инструкция для разработчиков

Автоматический бот ревью PR через ИИ (Феникс/Qwen). Проверяет Perl-код на ошибки, безопасность и стайлгайд.

---

## Как это работает

1. Ты создаёшь/обновляешь Pull Request в Bitbucket
2. Бот получает уведомление, скачивает diff
3. Отправляет код ИИ на анализ
4. Через 10–30 секунд в PR появляются комментарии с замечаниями

---

## Как подключить бот к своему репозиторию

### Шаг 1. Зайди в настройки репозитория

1. Открой свой репозиторий в Bitbucket
2. Перейди в **Settings** (шестерёнка слева)
3. В разделе **WEBHOOKS** нажми **Create webhook**

### Шаг 2. Настрой Webhook

| Поле | Значение |
|---|---|
| **Name** | `JARVIS PR Review` |
| **URL** | `https://pr-review-bot.bc-dev2.int.gazprombank.ru/webhook` |
| **Status** | ✅ Active |
| **SSL verification** | ❌ Skip (если есть такая галочка) |

### Шаг 3. Выбери события (Triggers)

Отметь **только** эти два события:

- ✅ **Pull Request: Opened** (срабатывает при создании PR)
- ✅ **Pull Request: Modified** (срабатывает при пуше новых коммитов в PR)

Остальное **НЕ** отмечай.

Нажми **Save**.

---

## Как пользоваться

Просто **создай PR как обычно**. Бот подключится автоматически и напишет комментарий.

**Что ты увидишь в PR:**
- 🔴 Ошибки (SQL-инъекции, отсутствие валидации)
- 🟡 Предупреждения (отсутствие eval/die, плохие практики)
- 💡 Советы по улучшению кода
- Итоговую сводку с количеством замечаний

### Если бот молчит

- Подожди 30 секунд (большой diff может проверяться дольше)
- Если прошло 2 минуты и ничего нет — нажми **Update** в PR (просто сделай ещё один коммит, даже пустой)

### Если бот пишет «Упс! Мой мозг не ответил»

ИИ временно недоступен. Это не ошибка PR. Попробуй позже или обновить PR.

---

## Чего бот НЕ делает

- ❌ Не пушит код в репозиторий (только читает)
- ❌ Не мержит и не отклоняет PR
- ❌ Не блокирует merge (это advisory, а не gate)
- ❌ Не заменяет ревью сеньора

## Чего бот НЕ проверяет

- Форматирование (пробелы, отступы) — это дело линтера
- Логику бизнес-правил — только технические ошибки
- Комментарии и документацию (только код)

---

## Полезные советы

1. **Маленькие PR = лучше ревью.** Бот анализирует первые 300 строк diff. Если PR огромный, часть кода может быть обрезана.
2. **Стайлгайд.** Бот проверяет по правилам команды. Если стайлгайд обновился — бот подхватит его автоматически.
3. **Ложные срабатывания.** ИИ может ошибаться. Если замечание неверное — просто проигнорируй его.
4. **Обновление PR.** Каждый новый пуш в PR триггерит повторную проверку.

---

## Если что-то сломалось

Напиши команде сопровождения. Для диагностики понадобятся:
- Скриншот вебхука из настроек репозитория
- Ссылка на PR

## Как обновить бота

Бот запущен на сервере `openWebUI` через Docker. Обновление — **без флешек, через репо**:

```bash
# 1. Зайти на сервер (через WinSCP или SSH)
ssh user@openwebui-server

# 2. Скачать свежий код из репо
git pull

# 3. Пересобрать образ и перезапустить
docker build -t pr-review-bot .
docker-compose up -d
```

Базовый образ Python и pip-пакеты подтягиваются из Nexus банка — интернет не нужен.

## Частая команда (одной строкой)

```bash
git pull && docker build -t pr-review-bot . && docker-compose up -d
```

## Webhook

После перезапуска URL бота не меняется: `https://pr-review-bot.bc-dev2.int.gazprombank.ru/webhook`.
Webhook в Bitbucket перенастраивать не нужно — он привязан к URL, а не к версии бота.
