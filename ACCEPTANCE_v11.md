## 1. REV-004 — стайлгайд не в UTF-8 НЕ роняет ревью

> Раньше: не-UTF-8 стайлгайд → «🤖 Внутренняя ошибка» в PR. Теперь должен прочитаться.

- [ ] Сохрани рабочий стайлгайд: `cp styleguide.md styleguide.md.bak`
- [ ] Сделай заведомо НЕ-UTF-8 файл (cp1251) с кириллицей. ⚠️ НЕ конвертируй
  `styleguide_unicorn.md` — в нём эмодзи 🦄, который в cp1251 не кодируется
  (`iconv: illegal input sequence`). Бери стайлгайд только с кириллицей, либо `//IGNORE`:
  ```bash
  iconv -f utf-8 -t cp1251//IGNORE styleguide.md -o styleguide.cp1251 && mv styleguide.cp1251 styleguide.md
  ```
  (на боевой VM реальный `styleguide.md` и так в cp1251 — REV-004 это уже подтвердил.)
- [ ] Открой любой маленький тестовый PR.
- [ ] В логах ожидаем **предупреждение, но НЕ падение**:
  ```
  ⚠️ Стайлгайд не в UTF-8 — прочитан как cp1251. Пересохрани styleguide.md в UTF-8.
  ```
  И ревью должно пройти (НЕ должно быть `❌ Ошибка ревью ... codec can't decode`).
- ✅ **Успех:** ревью отработало, в PR пришёл комментарий (а не «Внутренняя ошибка»).

## 4. cached_tokens — кэшируется ли префикс

- [ ] На любом PR из шагов выше глянь лог-строку:
  ```
  📥 Ответ Феникса: ... prompt_tokens=NNNN, cached_tokens=MMMM, completion_tokens=...
  ```
- Интерпретация:
  - `cached_tokens` > 0 (близко к `prompt_tokens`) → префикс/стайлгайд берётся из кэша →
    повторы между файлами почти бесплатны → **экономить не нужно**.
  - `cached_tokens=0` или `?` → кэша нет → есть смысл подумать про system/user-split (но это
    ~2% бюджета, не срочно).
